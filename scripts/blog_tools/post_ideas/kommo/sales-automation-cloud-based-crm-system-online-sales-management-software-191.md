Se você deseja receber informações mais precisas via webhook, é melhor criar Webhooks na seção de integrações.
Para começar, vá para Configurações e depois para a seção Integrações do Kommo. Nesta seção, você tem suas integrações instaladas. Você verá o botão "Webhooks". Clique nele, a janela modal será aberta:
Por favor, clique em "+ Adicionar webhook". Você precisará indicar o URL que receberá este webhook e o evento que enviará esse webhook
Os eventos são:
- Lead adicionado, editado, excluído, restaurado, usuário responsável alterado, status alterado, observação adicionada cada como evento separado
- Contato adicionado, editado, excluído, restaurado, usuário responsável alterado, observação adicionada como evento separado
- Empresa adicionada, editada, excluída, restaurada, usuário responsável alterado, observação adicionada como evento separado
- Tarefa adicionada, editada, excluída, usuário responsável alterado cada um como evento separado
- Usuário adicionado, editado, excluído, usuário responsável alterado, nota adicionada como evento separado
- Lead de entrada adicionado, editado, excluído, cada um como evento separado
- Mensagem recebida recebida
- Elemento adicionado, elemento editado ou excluído para qualquer uma das listas, cada um como evento separado
Você receberá informações completas sobre o evento que escolher, quando ocorrer, e poderá escolher vários eventos para acionar, por exemplo, vamos dar uma olhada no Lead editado
Quanto ao gatilho de webhook, você receberá informações para dois campos:
- No campo do lead: ID do lead, nome, ID do estágio, ID do estágio anterior, preço, ID do usuário responsável, data da última modificação, ID do usuário modificado, ID do usuário criado, ID do usuário criado, data de criação, ID do pipeline, tags do lead com seus IDs , e nomes, e qual campo foi alterado pelo nome do ID e como foi alterado com valores novos e antigos
- No campo Da conta: subdomínio, ID e o link da sua conta
