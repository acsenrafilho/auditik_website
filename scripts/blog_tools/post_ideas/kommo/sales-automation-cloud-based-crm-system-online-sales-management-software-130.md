Si desea recibir información más precisa a través de webhook, es mejor crear Webhooks en la sección de integraciones.
Para comenzar, vaya a Ajustes, luego a la sección Integraciones de Kommo. En esta sección, tiene sus integraciones instaladas, verá el botón "Enlaces web". Haga clic en eso, se abrirá una ventana modal:
Haga clic en "+ Agregar webhook". Deberá indicar la URL que recibirá este webhook y el evento que empujará este webhook
Los eventos son:
- Prospecto agregado, editado, eliminado, restaurado, usuario responsable cambiado, estado cambiado, nota agregada, cada uno como evento separado
- Contacto agregado, editado, eliminado, restaurado, usuario responsable cambiado, nota agregada, cada uno como evento separado
- Empresa agregada, editada, eliminada, restaurada, usuario responsable modificado, nota agregada, cada uno como evento separado
- Tarea agregada, editada, eliminada, el usuario responsable cambiado, cada uno como evento separado
- Cliente agregado, editado, eliminado, usuario responsable cambiado, nota agregada, cada uno como evento separado
- Prospecto entrante agregado, editado, eliminado, cada uno como evento separado
- Mensaje entrante recibido
- Elemento agregado, elemento editado o eliminado para cualquiera de su lista, cada uno como evento separado
Recibirá información completa sobre el evento que elija, cuando ocurra el evento, y puede elegir múltiples eventos para activar, por ejemplo, echemos un vistazo a Prospecto editado
En cuanto al activador webhook, recibirá información para dos campos:
- Desde el campo de prospecto: ID del prospecto, nombre, ID de etapa, ID de etapa anterior, precio, ID de usuario responsable, última fecha de modificación, ID de usuario modificado, ID de usuario creado, fecha de creación, ID de embudo, etiquetas del prospecto con sus ID, nombres, qué campo se cambió por nombre de ID y cómo se cambió con valores nuevos y antiguos
- Desde el campo de la cuenta: subdominio, ID y el enlace de su cuenta
